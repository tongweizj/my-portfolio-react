重构 React 前端时，核心逻辑是从 **页面导向”** 转向 **“数据与组件导向”**。

## 目录结构：按模块拆分

建议采用 **Feature-based Structure**，这与你后端的聚合路由逻辑是对应的：

```text
src/
├── api/              # axios 实例、拦截器配置
├── assets/           # 静态资源（图片、全局字体等）
├── components/       # 【通用 UI 组件】不带业务逻辑，如 Button, Input, Modal
├── features/         # 【业务核心】按模块聚合逻辑
│   ├── auth/         # 登录、注册、AuthContext、鉴权 Hook
│   ├── articles/     # 文章列表、编辑器、文章专用 API 请求
│   └── site/         # 站点配置、页头页脚信息
├── hooks/            # 全局通用的自定义 Hooks
├── layouts/          # 布局组件（如带侧边栏的布局、全屏布局）
├── pages/            # 路由页面（仅作为容器，组合 features 里的组件）
├── store/            # 全局状态管理 (Zustand/Redux)
└── utils/            # 通用工具函数（格式化日期等）

```

## 💡 为什么这样重构？

因为你后端现在返回的是干净的 **JSON + 状态码**：

1. 前端拿到 `401` -> 状态设为 `null` -> 路由守卫自动跳到登录。
2. 前端拿到 `200` -> 存入 `user` 状态 -> 全局 `Header` 自动显示“欢迎，XX”。
