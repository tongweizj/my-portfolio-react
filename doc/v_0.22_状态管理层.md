## 2. 状态管理层：从“碎片化”到“集中化”

- **旧逻辑**：

  每个页面都用 `useState` 存用户数据。

- **重构逻辑**：

  使用 **React Context API** 或 **Redux/Zustand** 建立一个 `AuthContext`。

- **应用**：

  你的 `verifyCookie` (或 `me`) 接口应该在应用根部调用一次，并将用户信息共享给 `Header`、`Profile` 等所有组件。

2. 建立身份中枢 (features/auth/)
   动作：创建 AuthContext.js。

目的：解决你之前提到的 me 接口调用问题。在应用启动时，由这个 Context 去问后端：“用户是谁？”

关联：这决定了导航栏显示“登录”还是“头像”。

## 🚀 针对你目前情况的第一个重构动作

建议你先重构 **`AuthContext.js`**。

你之前问过 `welcome` 和 `me` 的区别，在 React 重构中，你应该在 `AuthContext` 初始化时调用 `/api/users/me`：

```javascript
// src/features/auth/AuthContext.js
export const AuthProvider = ({ children }) => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // 应用启动时自动检查登录状态
    const checkStatus = async () => {
      try {
        const res = await axios.get('/api/users/me');
        setUser(res.data.payload); // 对应你后端 isSignedIn 返回的 payload
      } catch (err) {
        setUser(null);
      } finally {
        setLoading(false);
      }
    };
    checkStatus();
  }, []);

  return <AuthContext.Provider value={{ user, setUser, loading }}>{children}</AuthContext.Provider>;
};
```
